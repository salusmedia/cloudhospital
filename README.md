> 本仓库是 **AI 云医院** 全部应用场景的单体仓库（Monorepo）。

## 2. 一分钟跑起来

```bash
# 1) 安装依赖（前端 + 后端）
pnpm install                 # 安装所有 JS 依赖
pnpm run setup:py            # 为各 Python 服务创建虚拟环境并安装依赖

# 2) 启动本地全栈（含数据库、网关、所有已注册场景）
pnpm run dev

# 3) 只启动你负责的某个场景（例：场景 001）
pnpm run dev --filter=scenario-001-*
```

## 3. 核心约定（必须遵守）

1. **一个场景 = 一个编号目录**，前后端成对：`apps/scenario-XXX-frontend` + `services/scenario-XXX-backend`。
2. **跨场景共享的东西一律进 `packages/`**，禁止场景之间直接 import 对方代码。
3. **每个目录都有 `CLAUDE.md`**，是该模块的"宪法"，Claude Code 和人都按它来。
4. **主分支永远可部署**，所有改动走 `feature/scenario-XXX-*` 分支 + Merge Request。
5. **数据合规第一**：患者数据不出院内、不进日志、不进提交记录。详见各 CLAUDE.md 的合规条款。
